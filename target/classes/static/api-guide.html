CamCheck API Guide

This guide provides information on how to use the CamCheck API. For a complete and interactive API documentation, visit the Swagger UI.

Note: All API endpoints require authentication unless otherwise specified.

Base URL

The base URL for all API endpoints is:

https://your-domain.com/api/v1

Authentication

The API uses Basic Authentication. Include your username and password with each request.

Endpoints

GET /status

Get the current status of the camera system.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "streaming": true,
    "motionDetection": true,
    "recording": false
  }
}

POST /camera/stream/start

Start camera streaming.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "streaming": true
  }
}

POST /camera/stream/stop

Stop camera streaming.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "streaming": false
  }
}

GET /camera/snapshot

Take a snapshot from the camera.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "image": "base64_encoded_image_data..."
  }
}

PUT /motion-detection/{enabled}

Enable or disable motion detection.

Path Parameters

Parameter Type Description
enabled boolean Set to true to enable, false to disable

Response

{
  "status": "success",
  "message": null,
  "data": {
    "motionDetection": true
  }
}

POST /recording/start

Start recording video.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "recording": true
  }
}

POST /recording/stop

Stop recording video.

Response

{
  "status": "success",
  "message": null,
  "data": {
    "recording": false
  }
}

Error Handling

In case of an error, the API returns a response with status "error" and an error message:

{
  "status": "error",
  "message": "Error description",
  "data": null
}

Example Usage

Example using curl to get camera status:

curl -X GET "https://your-domain.com/api/v1/status" -u username:password -H "accept: application/json"

For a complete and interactive API documentation with all available endpoints, request/response schemas, and testing capabilities, visit the Swagger UI.
